## 文档的阅读顺序

1. 先跑一遍 [Quick Start](/docs/quick-start)，确认 `doctor`、`devices list` 和 `test event` 都有输出。
2. 再看 [Configuration](/docs/configuration)，把 `devices`、`actions` 和 `profiles` 的关系理顺。
3. 如果你主要依赖手势按钮，接着看 [Gestures](/docs/gestures)。
4. 真机行为不稳定时，直接跳到 [Troubleshooting](/docs/troubleshooting)。

<AgentCallout title="把 Agent 工作流放在后面一点接入" eyebrow="Agent-first">
  建议先确认系统动作和快捷键路径稳定，再把手势接到 AI 或 Agent 命令。这样排错会简单很多。
</AgentCallout>
